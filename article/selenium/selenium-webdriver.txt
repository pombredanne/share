[quote,SeleniumHQ,'http://seleniumhq.org/projects/webdriver/[Selenium WebDriver]']
________________________________________________________________________________
The biggest change in Selenium recently has been the inclusion of the WebDriver API. Driving a browser natively 'as a user would' either locally or on a remote machine using the Selenium Server it marks a leap forward in terms of browser automation.
________________________________________________________________________________

link:SelWebDriverMigrate.html[從 Selenium 1 到 Selenium 2]，最大的改變就是 Selenium 2 將 link:../WebDriver/WebDriver.html[WebDriver] 整合進來。Selenium 2 的功能還在持續增加中，每個版次增加的功能可以參考http://seleniumhq.wordpress.com/category/releases/[官方網誌]。

Selenium 2/WebDriver 在架構上與 link:SelRC.html[Selenium 1/RC] 截然不同，尤其可以不需要有像 link:SelSvr.html[Selenium Server] 這樣居間做為 proxy 的角色，language binding 直接跟做為 browser 一部份的 browser driver 溝通：

 WebDriver API / Language Bindings -> Native Drivers / Browsers

官方直接提供有 Java、C#、link:SelWebDriverPython.html[Python] 跟 Ruby 的 language binding。

其中 language binding 角色類似於 Selenium 1/RC 的 client library/driver，

比較特別的是，各 browser 的 driver 並非由 Selenium 團隊開發，而是由 browser 廠商自己依照 WebDriver API 提供不同的實作，因此 WebDriver 嚴然已經成為 link:../Browser/BrowserAuto.html[browser automation] 的新標準：

 * link:SelWebDriverFirefox.html[Firefox]
 * link:SelWebDriverOpera.html[Opera]
 * link:SelWebDriverIE.html[IE]
 * link:SelWebDriverChrome.html[Chrome]
 * link:SelWebDriverIOS.html[iOS]
 * link:SelWebDriverAndroid.html[Android]
 * link:SelWebDriverHtmlUnit.html[HtmlUnit]

WebDriver 可以從 OS level 發出 native event，這跟過去 Selenium 1.0 用 JavaScript 模擬出使用者行為的做法大不相同。

Selenium Server 還在，主要是用來搭配 Selenium Grid 做 distributed testing，但 Selenium Grid 也支援透過 WebDriver API 來操作（這跟 RemoteWebDriver 有關嗎？）。

Selenium IDE 自 1.1.0 開始就支援兩種不同 API 的輸出，例如 Python 2 (WebDriver) 跟 Python 2 (Remote Control)

WebDriver (object-oriented) API 支援 Python、Ruby、Java 及 C# 這些 binding，分別由熟悉不同語言特性的開發人員設計，所以 API 用起來不會覺得不順手。

自從 http://seleniumhq.wordpress.com/2011/07/18/selenium-2-1-released/[Selenium 2.1] 之後，就採用 weekly release。

 * http://nuget.org/List/Packages/Selenium.WebDriver[NuGet gallery] - .NET bindings

Release Notes:

 * http://code.google.com/p/selenium/source/browse/trunk/java/CHANGELOG[Java]
 * http://code.google.com/p/selenium/source/browse/trunk/py/CHANGES[Python]
 * http://code.google.com/p/selenium/source/browse/trunk/rb/CHANGES[Ruby]
 * ...

各 browser 以不同的方式提供 driver。
