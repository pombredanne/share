Appium / Android / Getting Started

  1. 透過 npm 安裝 Appium。

  2. 啟動 Appium server。 

    $ appium
info: Welcome to Appium v0.10.1 (REV d0827d6b2507c982802d02b2c6d2bb4d6906bdb3)
info: Appium REST http interface listener started on 0.0.0.0:4723
   info  - socket.io started

  3. 從 source 取得 example tests，這裡以 Python 為例。

    sample-code/examples/python/android.py
    import os
from time import sleep

from selenium import webdriver

# Returns abs path relative to this file and not cwd
PATH = lambda p: os.path.abspath(
    os.path.join(os.path.dirname(__file__), p)
)

desired_caps = {} # 
desired_caps['device'] = 'Android' # 
desired_caps['browserName'] = ''
desired_caps['version'] = '4.2'
desired_caps['app'] = PATH(
    '../../../sample-code/apps/ApiDemos/bin/ApiDemos-debug.apk' # 
)
desired_caps['app-package'] = 'com.example.android.apis'
desired_caps['app-activity'] = '.ApiDemos'

driver = webdriver.Remote('http://localhost:4723/wd/hub', desired_caps) # 

print driver.get_window_size()
elem = driver.find_element_by_name('Graphics') # 
elem.click()
elem = driver.find_element_by_name('Arcs')
elem.click()
driver.quit()
    除了 desired capabilities 之外，其實看不出來跟一般 Selenium 2/WebDriver 的測試有何不同…
    ApiDemos-debug.apk 是透過 source code 安裝才有的。
    透過 Remove WebDriver 跟 Appium server 溝通。
    如果要執行在 Android 4.1+ 以前的環境，則要改用 Selendroid Mode。
    透過 find_element[s]*() 定位到要操作的對象（WebElement），就可以對它進行操作。

  4. 將 API 16+ 的 emulator 或實體機準備好。執行測試： 

    $ python android.py
{u'width': 1280, u'height': 720}

Appium 預設採用 fast reset，如果啟動 Appium server 時有加 --full-reset 或 --no-reset，這裡就會是 false。
事先將 SUT 的 .apk 及 clean.apk 都用 debug key 做 sign 的動作，接下來如果判定需要清 app data 時，clean.apk 才能清掉資料。
"Running fast reset clean:" 這一行只有真正需要透過 clean.apk 來清 app data 時才會出現。
顯然 --full-reset 會在 session 開始跟結束時，各執行一次移除 app 的動作。
內部透過 activity manager 直接叫出 desired capabilities 裡指定的 launcher activity。
