development.html[Python / Development] / Project Structure

下面以 MyCalc 專案為例：

MyCalc/
|-- mycalc/  
|   |-- calc.py
|   ｀-- __init__.py
|-- setup.py
|-- setup.cfg
|-- README
|-- INSTALL
|-- LICENSE
|-- docs/    
|-- scripts/ 
`-- tests/   
    |-- __init__.py
    `-- unit/
        |-- __init__.py
        ｀-- test_calc.py
主要的程式碼都會集中放在一個資料夾底下，形成一個 package。例如這裡的 mycalc 就是 package name。
docs/ 用來擺放所有的文件。
scripts/ 用來擺放使用者會直接調用的 CLI 程式（如果有的話），例如 batch file 或 shell script 等。
測試程式都擺在 tests/ 底下，也可以用子資料夾將不同類型的測試區分開來。

搭配 nose 使用時，unit/__init__.py 的存在很重要，因為 否則 nosetests 就不會遶進去找到 test_calc.py。
