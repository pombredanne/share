Python Programming: 當 Module 越來越大…

  1. Module 可以 import 很多組成進來，從 module 使用的角度來看可能有很多東西，但不一定要寫成一個檔案。helper module

    假設一開始 foobar.py 還很單純，寫在同一支 .py 檔裡還容易維護。

before/foobar.py
def foo(): print 'foo'
def bar(): print 'bar'

用起來像是這個樣子：

before/client.py

from foobar impor *
foo(); bar()

之後 module 越來越大時，可以拆出 package，但不影響原有的用法…

after/foobar/__init__.py
from foo import *
from bar import *
after/foobar/foo.py
def foo(): print 'foo'
after/foobar/bar.py
def bar(): print 'bar'

用 private module 也是可行的方式之一？可是檔名前面加個底線好像也可以？或許拆出 package 再將拆細的 private module 前面冠上底線是不錯的方式？

module 太大，檔案上方會有一堆不知道是用在哪裡的 import…
