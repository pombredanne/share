:title: [Jython] Registry

Jython 的行為會受到一些 properties 的影響，而 properties 的來源可以是：（按 priority 由低至高排列）

 1. link:../java/system-properties.html[Java system properties]
 2. Jython system-wide registry - 預設是 `$JYTHON_HOME/registry`。
 3. User's personal registry - 預設是 `~/.jython`。
 4. Command-line options - 用 `-D<name>=<value>` 給定的 Jython properties。

常用的有：

 * `python.path` - 等同於 link:pythonpath.html[CPython 的 `PYTHONPATH`]。
 * `python.cachedir` - 

雖然 http://manpages.ubuntu.com/manpages/precise/man5/jython.conf.5.html[Ubuntu Manpage: jython.conf] 有這麼一段話 "However, since the jython and jythonc startup scripts take responsibility for starting the Java interpreter, it is unlikely that you will use this method of Jython configuration."，但還是難以參透 Jython properties 跟 Java system properties 有什麼不同？

不過 http://www.jython.org/javadoc/org/python/core/Options.html[`org.python.core.Options` 的 Javadoc] 有這麼一段話 "A class with static fields for each of the settable options. The options from registry and command line is copied into the fields here and the rest of Jython checks these fields."，儘管如此，好像最後也都能從 `System.getProperty()` 存取得到...

--------------------------------------------------------------------------------
$ grep -E -v '^#.*' registry | grep -E -v '^$'
python.cachedir = cachedir
python.packages.paths = java.class.path, sun.boot.class.path
python.packages.directories = java.ext.dirs
python.security.respectJavaAccessibility = true
python.options.caseok = false
$
$ jython -Dfoo=bar
>>> from org.python.core import Options
>>> Options.caseok
False
>>> from java.lang import System
>>> System.getProperty('foo')
u'bar'
>>> System.getProperty('python.packages.paths')
u'java.class.path, sun.boot.class.path'
>>>
