AsciiDoc / Configurations / Document Attributes

document attribute 是 texual name-value pair，大量被用在組態檔內做條件式輸出（conditional markup generation）或文字置換（textual substitution），當然也可以用在一般的文件裡。例如要讓文件出現小圖示，可以在文件裡頭宣告 :icons: 或是從命令列指定 -a icons，就是 document attribute 的一種應用。

attribute name 不分大小寫，只能由英數字、減號（-）跟底線（_）組成。無論 attribute value 為何，attribute 至少有 defined 跟 undefined 兩種區別，就算 attribute value 是空字串，也算是 defined。

document attribute 可以從幾個不同的地方定義（define），分別具有不同的優先權：（由高至低）

  1. 命令列（command line）。

  2. 文件內的 attribute entry、attribute list、block ID。

  3. 組態檔內的 [attributes]。

同一個 attribute 如果被 define 多次，以優先權（precedence）較高者為主，如果優先權相同，則以最後 define 者為主。

$ echo -e ":who: World\nHello, {who}" | asciidoc -s - 
<div class="paragraph"><p>Hello, World</p></div>
$
$ echo -e ":who: World\nHello, {who}" | asciidoc -s -a who=AsciiDoc - 
<div class="paragraph"><p>Hello, AsciiDoc</p></div>
:who: World 是在文件內定義 attribute 的寫法，下面有更完整說明。至於 {who} 的表示法可以做文字代換，更多 attribute 的用法可以參考 Attribute References。
從 command line 提供的 attribute value 會覆寫在文件內的定義，所以輸出 "Hello, AsciiDoc"。
