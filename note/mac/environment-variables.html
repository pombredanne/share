Mac OS X: 設定環境變數（Environment Variables）

基於 Unix 的 Mac OS X，當然也支援 ~/.bash_profile 的用法。但透過這種方式設定的環境變數，只有 (thir-party) Terminal app 可以套用得到，OS X 視窗應用程式則完全不理會這裡的設定。

儘管如此，實務上還是很常將環境變數設定在 ~/.bash_profile 裡，因為 OS X 視窗應用程式通常不會用到環境變數。

如果環境變數要同時讓 Terminal app 及其他 OS X 視窗應用程式套用到，就必須要設定在 ~/.MacOSX/environment.plist，這個檔案只有在使用者登入時才會被讀取，所以在裡面所做的異動要重新登入才會套用。

這個檔案一開始並不存在，可以用下面的方法建立：

$ cd
$ mkdir .MacOSX
$ echo "" > .MacOSX/environment.plist 
$ open .MacOSX/environment.plist
這個動作很重要，如果單純用 touch 來產生這個檔案的話，下一步用 XCode 開啟檔案時就會遇到 "The data couldn’t be read because it has been corrupted." 的錯誤。
用 XCode 開啟編輯。
