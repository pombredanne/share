= link:selenium.html[Selenium] / UI Mapping =

簡單的說，UI Mapping 的概念就是用一個 UI Map 來集中管理重複／散落在各處的 link:selenium-locator.html[locator]。可以帶來下面幾個好處：

 * 降低 test script 的維護成本；一旦有 UI 調整而導致 locator 失效的狀況，只要調整一個地方即可。
 * 提高 test script 的可讀性；用邏輯名稱來置換 locator 的表示法，例如用 `nav_help_about_link` 來取代 `"//*[@id='ddm_help']/li[5]/a"`。

基本上 UI Mapping 也算是個 link:../design/design-pattern.html[Design Pattern]，只要掌握上述的精神，實作上有很多可能性，包括抽出成可以共用的變數、用外部的文字檔統一存放所有的 locator 等...

NOTE: 全面採用 link:selenium-page-object.html[Page Object Pattern] 的話，link:../seleniumlibrary/seleniumlibrary.html[SeleniumLibrary] 會完全無用武之地，UI Mapping 是個很好的折衷。要如何讓 Page Object Pattern 跟 UI Mapping 兩者完美地結合？
