Selenium Locators: CSS

css=<css_selector>

簡單說明一下 CSS selector 的語法：

CSS 用來套用 Styles 的 Selector 也可以用來定位 Element, 比較不一樣的地方是該 Selector 必須只定位到單一個 Element 如果不是互動性很高的頁面, 通常不太會去標 id, 再加上 CSS 比 XPath 快, 有些時候 CSS Locator 可能是最好的選擇 (但如果互動性很高, class 通常會不斷產生變化, 因此不適合做為判斷的依據; 更何況美工人員也不覺得變換 class 名稱會有什麼影響…) Examples css=form#loginForm css=input.required[type="text"] css=div.logon_portal strong span ←- 中間有空格沒關係 HTML 的 class 屬性可以給定多個 Class, 比對時比較常用 [contains(@class, class_name)] 的方法, 否則前後多個空白就比對不到了

不要用 XPath 來取代 CSS strategy：（但如果是 substring 的比對則非 XPath 不可）

  • XPath: //div[contains(@class, article-heading)]

  • CSS: css=div.article-heading
