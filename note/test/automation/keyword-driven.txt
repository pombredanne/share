= Keyword-Driven Automation Framework =

[quote,'http://sourceforge.net/projects/safsdev/[Software Automation Framework Support | SourceForge.net]']
________________________________________________________________________________
 * Tool-independent implementation for long-term test automation success.
 * Tool-independent test design allowing tool migration and coop execution.
________________________________________________________________________________

 * Keyword = abstraction of layers
 * Division of Labor - Test Designer and Test Engineer
 * Keyword-Driven 的精神是寫出來的 test case 也要可以手動跑，所以第一層用口語化的方式描述是有必要的。
 * 區分 Planning/Implementation Stage 兩個階段的做法是關鍵，至少它強調了 keyword 的 input/output 是需要被 design 過的。
 * Plan 也意謂著，結合不同 scenario 的需求，規劃出重用性較高的 keyword。
 * KW 也可以把 tricks 包裝起來
