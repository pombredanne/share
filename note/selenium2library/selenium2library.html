Selenium2Library

為了提高 test data 的穩定性，keyword 應該只描述某個步驟的意圖（intent），而不應該透漏出底層的做法，否則未來工具或做法有異動時，test data 就得要一併做調整，無形間增加了維護的成本。因此強烈建議不要採用 Selenium2Library 這類簡單包裝過 testing tools 的 test libraries，而應該直接用 testing tools 實作 domain-specific keywords，可以參考 Page Object Pattern 的做法。

如果很不幸地，你已經被 SeleniumLibrary 綁住，還好 Selenium2Library 一開始就考量到從 SeleniumLibrary 遷移過來的問題，keywords 幾乎是保留舊有的用法（重新用 Selenium 2/WebDriver 實作過），所以 test data 需要調整的地方並不多…

還好！還好 Selenium2Library 想得這麼周到，又幫我們爭取到了一點時間，可以脫離 Selenium(2)Library 的束縳…
