:title: [Git] Repository Hosting

給別人代管：

 * link:../github/github.html[GitHub] - 無限制的 public repositories，同時提供有 per-user/project pages (link:../github/github-pages.html[GitHub Pages])，有助於好的 first impression 跟專案的推廣。
 * https://bitbucket.org/[Bitbucket] - 無限制的 private repositories，只有 per-user pages，對專案的行銷沒有太大幫助。

自行管理：

 * link:dropbox-remote.html[Dropbox] - 用 Dropbox 做 Git Remote。
 * http://gitlabhq.com/[GITLAB] - 自行架設 server，適合需要保護 source code 的商業公司。
 * https://gitorious.org/[Gitorious]

 * http://stackoverflow.com/questions/109440/best-git-repository-hosting-for-a-commercial-project[Best Git repository hosting for a commercial project? - Stack Overflow] (2008-09-20) - GitHub 的 private repository 不算便宜。Toby 提到 "the tendency here is to many small repositories, with few collaborators"，或許 Bitbucket "unlimited private git repositories with up to five users" 的訴求對 private/commercial startup 比較實用。也有人提到 "business relies heavily on your source-code" 應該要自架 server。
