Git / Repository Hosting

給別人代管：

  • GitHub - 無限制的 public repositories，同時提供有 per-user/project pages (GitHub Pages)，有助於好的 first impression 跟專案的推廣。

  • Bitbucket - 無限制的 private repositories，只有 per-user pages，對專案的行銷沒有太大幫助。

自行管理：

  • Dropbox - 用 Dropbox 做 Git Remote。

  • GITLAB - 自行架設 server，適合需要保護 source code 的商業公司。

  • Gitorious

參考資料

文件

  • GitHub vs. BitBucket: Shifting Value Propositions | Stephan Sokolow’s Blog (2012-06-13) - 提到 "when people host a project, they actually care about getting potential users interested in it."，所以 GitHub Pages 的角色很重要。

其他文件

  • Best Git repository hosting for a commercial project? - Stack Overflow (2008-09-20) - GitHub 的 private repository 不算便宜。Toby 提到 "the tendency here is to many small repositories, with few collaborators"，或許 Bitbucket "unlimited private git repositories with up to five users" 的訴求對 private/commercial startup 比較實用。也有人提到 "business relies heavily on your source-code" 應該要自架 server。
