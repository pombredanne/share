Subclipse / 安裝

Subclipse 跟 Subversion Client API (JavaHL) 的版本綁得很緊，所以在決定 Subclipse 要安裝哪個版本之前，要先確認 JavaHL 在該作業系統下可以取得的最新版本為何。

  1. 安裝 JavaHL。

    要先安裝 JavaHL，否則 Eclipse 下的 SVN 還是會出現 JavaHL (JNI) Not Available 的錯誤提示。

    • 在 Windows 下，如果採用 32-bit JVM，只要安裝時有選取 Subversion JavaHL Native Library Adapter 即可。若是採用 64-bit JVM，官方建議可以藉由額外安裝 Silk SVN 來取得 64-bit 的 JavaHL。

    • 在 Ubuntu 下，則必須安裝 libsvn-java 套件（主要是安裝 /usr/lib/jni/libsvnjavahl-1.so），在 eclipse.ini 加上 -Djava.library.path=/usr/lib/jni，之後重新啟動 Eclipse 即可。

    • 在 Mac OS X 下，按官網的建議到 openCollabNet 下載安裝 Universal Subversion （安裝到 /opt/subversion），即可獲得 JavaHL 的支援。

  2. 新增一個 repository Subclipse Updates 指向 http://subclipse.tigris.org/update_1.6.x （需要 JavaHL 1.6.x） 或 http://subclipse.tigris.org/update_1.4.x （需要 JavaHL 1.5.x）。

  3. 安裝 Subclipse、Subclipse Client Adapter 跟 Subversion JavaHL Native Library Adapter（採用 64-bit JVM 也要勾選），另外 Subversion Revision Graph 也滿實用的。

    其中 Subversion JavaHL Native Library Adapter 做為 Subclipse Client Adapter 這個 common API 的一種實作。JavaHL 是一種 native library，所以這一段的問題會比較多…
