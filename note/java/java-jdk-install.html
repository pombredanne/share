[Java][JDK] 安裝

Windows

JDK 的安裝程式並不會自動更新 PATH 環境變數，為了方便執行到 javac.exe 與 javadoc.exe 等 JDK 開發工具，建議將 jdk<version>\bin 的完整路徑加到 PATH 環境變數裡。

官網說安裝過後要自己加 %PATH，但裝完 JDK 7 的結果發現 java.exe、javaw.exe 等都會出現在 C:\Windows\System32 裡（應該是從 C:\Program Files\Java\jre7\bin 複製出來的？），而這個路徑原本就在 %PATH% 裡，不用額外設定。

雖然 JDK 安裝完畢後，安裝程式會跳出來問要不要繼續安裝 JRE，但事實上，安裝 JDK 時，已經安裝了一份 JRE 在 jdk<version>\jre。這也就是為什麼，從程式集移除 JRE 後，JDK 還可以繼續運作的關係，靠的就是 JDK 內部私用的那一份 private JRE。相對於 private JRE，明確安裝的 JRE 則稱做 public JRE；private JRE 僅供 JDK tools 使用，而 public JRE 則供其他 Java application 使用。

JDK 裝完後還會跳出來問要不要裝 JavaFX SDK，可以選擇不要。

CentOS

CentOS 預設採用 OpenJDK，如果只是單純想要一個 Java 執行環境，安裝 OpenJDK 就夠了。

# yum install java-1.6.0-openjdk

還是想安裝 Sun/Oracle JDK 的話，必須要手動安裝才行：

  1. 從 Oracle 下載 JDK (*-rpm.bin)，例如 jdk-6u26-linux-x64-rpm.bin。

  2. 執行安裝程式 

    # chmod 755 jdk-6u26-linux-x64-rpm.bin
# ./jdk-6u26-linux-x64-rpm.bin
