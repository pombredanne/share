== 強哥 ==

 * Open Source 授權問題可以找強哥 （legist）
 * http://www.openfoundry.org/of/download/openlegal/slide_richard_code_licensing_issue_on_web_developing_at_PHPConf.tw_2011/phpconf_20111112_richard.pdf
 * Public Domain 也是一種授權（極致，最最最自由），例如 http://www.sqlite.org/copyright.html[SQLite]
 * Public Domain 無法達成四大自由
 * Copyleft，把 (c) 左右倒過，
 * AGPL (Affero) 適用 SaaS，因為 code 不會到前端。 --> 不修改就自由，修改就要釋出
 * 約束性 AGPL > GPL > LGPL
 * close source 避免被別人用（obfuscator 防君子不防小人），倒不如 open 出來，用 license 保證自己。
 * 先開個 open source project, 再回饋回公司。讚！
