Pip

Pip 跟 EasyInstall 一樣，可以用來安裝 PyPI 上的套件，官網明白提到它就是要來取代 EasyInstall 的。

它與 EasyInstall 確實存在一些差異，最明顯的就是它沒有 EasyInstall 不支援移除套件的問題。不過現階段而言，EasyInstall 還是有 Pip 無法取代的地方：

pip is meant to improve on easy_install. Some of the improvements:

  • All packages are downloaded before installation. Partially-completed installation doesn’t occur as a result. Care is taken to present useful output on the console.

  • Packages don’t have to be installed as egg archives, they can be installed flat (while keeping the egg metadata).

  • Uninstallation of packages.

  • …

pip doesn’t do everything that easy_install does. Specifically:

  • It cannot install from eggs. It only installs from source. (In the future it would be good if it could install binaries from Windows .exe or .msi – binary install on other platforms is not a priority.)

  • It is incompatible with some packages that extensively customize distutils or setuptools in their setup.py files.

— Relationship with other tools — pip documentation

除了取代 EasyInstall 之外，更重要的是它在 virtualenv 裡扮演了很重要的角色。
