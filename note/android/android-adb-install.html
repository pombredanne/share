透過 ADB 安裝／移除套件

如果手邊有 APK 檔，透過 install 這個 device command 就可以進行安裝：

$ adb shell <apk_file>
$ adb install Development.apk
2087 KB/s (112988 bytes in 0.052s)
        pkg: /data/local/tmp/Development.apk
Success
$ echo $?
0
$
$ adb install Development.apk
2087 KB/s (112988 bytes in 0.052s)
        pkg: /data/local/tmp/Development.apk
Failure [INSTALL_FAILED_ALREADY_EXISTS] 
$ echo $?
0
重覆安裝同一個套件會失敗（就算是版本新一點，也不會有 "昇級" 的效果）。雖然 Failure […] 寫著失敗的原因，但並沒有在 exit status 反應出來。

透過 Activity Manager 可以直接叫出 Google Play 要安裝某個套件的畫面。以 aLogcat (org.jtb.alogcat) 為例：

$ am start market://details?id=org.jtb.alogcat
Starting: Intent { act=android.intent.action.VIEW dat=market://details?id=org.jtb.alogcat }

要移除套件的話，

$ adb uninstall <package_name>
$ adb uninstall com.android.development
Success
$ adb uninstall com.android.development
Failure

在這個過程中，都可以用 Package Manager 先確認一下該套件是否已經安裝或移除了：

$ pm list packages
package:android
package:android.tts
package:at.abraxas.amarino
...

參考資料

  • Install Application programmatically on Android - Stack Overflow (2011-01-05) - 提到直接下載 apk 或開啟 market 安裝某個套件的 intent。

  • Owen’s: /Android/ Install an App in Android (2011-02-21) - 除了 adb install 從外部安裝 apk 之外，還有 pm install 從內部安裝 APK 的選擇，也可以先放到 /data/app 或 /system/app，開機的時候會自動安裝？
