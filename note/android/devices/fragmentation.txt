Open Source -> 百化齊放 PK 蘋果一家獨大 -> 差異化／機海戰術 -> 多樣化／新鮮感 -> 市場繁榮 -> 有礙統一／進化
碎片化 -> 用戶體驗良莠不齊
Google 不強制廠商對舊產品提供升級支持（沒有新賣點時才會想到系統升級，硬體差異要開發不同驅動的成本不低，更新後出事反而更糟，減少購買新機的慾望；因此多數停留在購買時的版本），新版本不能迅速普及。常用的系統版本超過 10 個，
為了確保兼容性 -> 增加開發／測試成本

 -> 高／中／低階市場 -> 不同的硬體配置。
製造商貪心 -> 開發者分心 -> Android App 的精緻度通常不及 iOS App 的現象。

 * Model - 同一製造商可能為了市佔率，通吃高低階市場，導致硬體配置的組合很多樣。
 * Brand - 搭載自家的 Manufacturer Overlay/Skin（例如 HTC Sense 或 Samsung TouchWiz 等）。
 * API Level - 開發者很難取捨，要不要放棄舊 API 的使用者來換取新 API 的功能？ 
 * Resolution/Density - 這個問題對開發／測試的影響最大，產品的兼容性直接影響用戶體驗。
