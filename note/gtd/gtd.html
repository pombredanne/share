Getting Things Done (GTD)

一些想法：

  • 不適合將學習事項也加到 GTD 裡，否則很快就會失控，應該每天保留一段時間學習新的事物，至於學習進度的掌控則要利用其他的工具做管理。

    • 又或者將要學習的事項切細，確定這幾天要完成的部份才放到 GTD 追蹤。

  • 由於 "公事" 與 "私事" 都會匯集到一個 "個人", 因此 GTD 不用刻意去做區分.

  • GTD 跟專案管理是不衝突的, 即便已經有一份專案計劃, 個人還是必須依照自己的狀況擬定 "執行細節"

  • GTD 是用來做 Time Tracking 的地方, 按時間由近到遠來安排 log 是比較直覺的, 資料歸檔應該放到 PIM 裡.

  • 排除 Pending 的狀況, 否則就列入 Someday

  • 雖然 "行事曆" 與 "待辦事項" (Todo) 兩者本質上有 "特定時間要做" 與 "有時間就趕快做" (ASAP) 的差異, 但實務上發現 Todo 如果沒有善加管理, 很容易就會失控.

  • 建議給 Todo 一個 "時段" 的約束, 而非放任它做為 ASAP, 否則短期間就會累積成 "不知道該從哪裡下手" 的無效清單.

  • 每天持續追蹤才用 @wait; 儘可能跟對方確認完成時間, 因為 @wait 有結束時間時, 會輸出到 "今日事今日畢" 裡, 時效的掌控度會比較好…

  • 當無法決定處理的時間時, 建議將時間調整為 "明天/下星期一/下個月初", 方便以 "隔天/一個星期/一個月" 做為檢討的週期

  • ASAP 某種程度上 unscheduled 是一樣的意思, "今天" 的行事曆不該出現 ASAP 的項目, 而應該在前一天調整過, 如此當天沒有被執行到時才會納入 "昨天" 做為提醒

  • 前一天應該將明天所有工作事項排定精確的 "時間點" 或 "時段", 如此才能妥善地分配有限的工作時間 (越接近要執行的日期, 時間就會越精確)

  • 儘可能將一件事情切割成可以完成或管理的步驟, 否則很容易被忽略而沒有進度; 心態上會變成 "今天沒做, 明天再說"

  • 將目前在進行的工作標示為 @focus, 開始進行前要先規劃好幾個步驟, 如此才能逐步地完成一項工作; 被標示為 @focus 通常還不夠細, 常常回過神來時, 看著 focus 的項目, 還是抓不到現在要做什麼的感覺…

  • 儘量少用週期性的標示, 做完這一次就立刻將時間調往下一個時間點, 這樣才能做到 "定期檢討"!!

  • 最好維持每天回顧 GTD 的好習慣; 如果工作一忙而疏於維護, 導致 GTD 崩潰時, 切記要先專注在新進的項目上, 下班再找時間調整過期的項目. 否則很容易變成瞎忙 - 一直擔心舊的項目, 卻又沒有心思把新進的項目做好…

  • 零碎的工作最好在零碎的時間做, 如此才能空出連續的時間, 去做比較花心思的事; 例如上班的時候可以拿來學習工作上需要用到的新知, 休息時間 (08:30 09:00 及 17:30 18:00) 再去做零碎的處理.
