在 RTM 上實現自己的 GTD

  • 用 nnn 全數字的 tag 來實作 project 的概念，然後再用 [tag] 來細分出同一 project 底下的 tasks，這樣可以緩解 project 數量成長太快的問題。而所謂的 misc 指的應該就是不屬於某個 project 的部份。

  • 隨時確認自己現在在做的事情有沒有在 GTD 裡，如果沒有就補上，這樣才不會老是在忙表定以外的事。事實上，會隨手拿起來做的事，某種程度上表示它盤據你的心頭，如果不放到 GTD 裡面追蹤怎麼行。

  • 所有的 item 都應該要有一個 due date。project 最長可以一個月，但 task 則不超過一個星期，目的就是為了提醒自己定期 review。所以如果有一個 project/task 是 2012-10-30 前要做完即可（現在是 2012-09-06），將 due date 設成 2012-10-30 並不妥當，因為可能到了 2012-10-30 前幾天才驚覺沒有時間了。為了避免這種狀況，可以在 project/task 的名稱及 due date 上動一些手腳，也就是將真正的到期日寫在名稱裡（"10/30 前完成 xxx"），但 due date 則設成距離現在不要太久的時間，提醒自己要做 review。
